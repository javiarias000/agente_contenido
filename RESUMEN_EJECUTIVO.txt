PROBLEMA A: Logo sale del frame (SEVERIDAD: ALTA)
┌─────────────────────────────────────────────────────────────────┐

┌─────────────────────────────────────────────────────────────────┐

│ Causa: Multiple: Whisper sin word-level timestamps,             │
│        path escaping incorrecto, silent fallback sin logging     │
│ Impacto: Video sin subtítulos, usuario no sabe por qué          │
│ Solución: Validación robusta + AdvancedSubtitleGenerator        │
└─────────────────────────────────────────────────────────────────┘

PROBLEMA C: Sin transiciones (SEVERIDAD: MEDIA)
┌─────────────────────────────────────────────────────────────────┐
│ Ubicación: skills/assembler.py línea 161-171                    │
│ Causa: ffmpeg concat demuxer (corte seco)                       │
│ Impacto: Transiciones abruptas entre escenas                    │
│ Solución: xfade filter con 200ms fade entre clips               │
└─────────────────────────────────────────────────────────────────┘

PROBLEMA D: VideoAnimator superficial (SEVERIDAD: BAJA)
┌─────────────────────────────────────────────────────────────────┐
│ Ubicación: skills/video_animator.py línea 155-197               │
│ Causa: Filtros ffmpeg estáticos (no keyframes), Google Veo      │
│        API no usada, post-processing del video completo         │
│ Impacto: Animación genérica y cara (re-encode)                  │
│ Solución: Integrar en Assembler con zoompan + keyframes         │
└─────────────────────────────────────────────────────────────────┘

PROBLEMA E: Contexto sin validación (SEVERIDAD: BAJA)
┌─────────────────────────────────────────────────────────────────┐
│ Ubicación: pipelines/base_pipeline.py línea 67                  │
│ Causa: No validación de outputs antes de mergear en contexto    │
│ Impacto: Silent failures si un step genera outputs inválidos    │
│ Solución: Type checking y validación en base_pipeline           │
└─────────────────────────────────────────────────────────────────┘

2. ARQUITECTURA PROPUESTA
================================================================================

COMPONENTES NUEVOS:

1. AnimatedImageGenerator (skills/animated_image_generator.py)
   ├─ Extiende ImageGenerator
   ├─ Extrae motion_hint del visual_description
   ├─ Guarda motion.json con metadatos de animación
   └─ Outputs: image_paths[], motion_metadata[]

2. AdvancedSubtitleGenerator (skills/advanced_subtitle_generator.py)
   ├─ Transcribe con Whisper (con validación)
   ├─ Fallback a script si Whisper falla
   ├─ Valida timings (no gaps, no overlaps)
   └─ Outputs: srt_path, srt_metadata

3. ComposedVideoAssembler (skills/composed_video_assembler.py)
   ├─ Reemplaza Assembler
   ├─ Compone escenas con:
   │  ├─ Imagen de fondo
   │  ├─ Animación con zoompan + keyframes
   │  ├─ Logo con safe area
   │  ├─ On-screen text overlay
   │  └─ Audio sincronizado (44100 Hz stereo)
   ├─ Concatena con transiciones (xfade)
   ├─ Quema subtítulos con validación
   └─ Outputs: final_video_path, srt_path

CAMBIOS EN EXISTENTES:

1. ugc_pipeline.py (línea 84-119)
   ├─ ImageGenerator → AnimatedImageGenerator
   ├─ Assembler → ComposedVideoAssembler
   └─ REMOVER: VideoAnimator (integrado en Assembler)

2. script_generator.py (línea 46-54)
   └─ Agregar motion_hint al prompt

3. video_animator.py
   └─ Deprecate (funcionalidad movida a ComposedVideoAssembler)

3. BENEFICIOS DE LA NUEVA ARQUITECTURA
================================================================================

✓ Logo siempre en frame con safe area (5% margin)
✓ Subtítulos robustos con validación y logging
✓ Transiciones suaves entre escenas (fade 200ms)
✓ Animación por escena, no post-processing
✓ Composición visual centralizada (overlays, texto, etc)
✓ Audio siempre 44100 Hz stereo
✓ Manejo de errores mejorado (no silent failures)
✓ Performance: menos re-encoding (animación integrada)
